How it works
01

Define catalog

Set the guardrails. Define which components, props, and actions the AI can use. Every interface is constrained to your design system.

02

AI generates

Describe what you want. Claude generates a JSON spec constrained to your catalog. Exact + semantic caching means repeat requests are 0ms.

03

Stream & render

Components render progressively as JSON streams in. Refine with patches instead of regeneration. Rate limiting protects your API budget.

Why spec-forge

json-render calls the LLM on every request. spec-forge adds a production layer.

json-render (direct) spec-forge
Cache None SHA-256 exact + TF-IDF semantic
Repeat request 3-5s LLM call 0ms
UI update Full regeneration Patch-based (Add/Replace/Remove)
Streaming Basic Incremental parser + element SSE
Rate limiting None Token bucket + concurrency semaphore
API key Client-side Server-side only
3D support 43 components (client-side) 43 components + server presets + 3D-specific validation
Catalog presets None 6 built-in (dashboard, form, ecommerce, 3D, etc.)
